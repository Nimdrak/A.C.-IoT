Release Notes for Version 6.0

In addition to many performance enhancements, Gurobi 6.0 adds several new features:

  • Distributed MIP solver: Gurobi users can now harness the power of multiple independent machines to solve a MIP model in parallel. You simply provide a list of machines that can be used as distributed workers, and Gurobi automatically distributes the work among the available machines. This new distributed MIP solver complements our existing distributed concurrent solver and our distributed tuning tool. New parameters have been added to support this new capability:
    • DistributedMIPJobs: Indicates how many distributed MIP jobs to start. These jobs run on distributed worker machines, which you set up before running the distributed MIP solver.
    • WorkerPool: A list of machines that can be used as workers for distributed algorithms.
    • WorkerPassword: The access password for the distributed worker machines.
    We've also extended our existing distributed concurrent solver to support both LP and MIP models (it previously only handled MIP models). The ConcurrentMIPJobs has been renamed to ConcurrentJobs to reflect the change. Please refer to the distributed parallel algorithm discussion in the Gurobi Reference Manual for additional details on all of our distributed algorithms.
  • Piecewise-linear objective support: You can now explicitly model piecewise-linear objective functions. This is done through the new GRBsetpwlobj function in C, and the new GRBModel.setPWLObj methods in the object-oriented interfaces. Piecewise-linear objectives are handled directly by the simplex algorithm, so if you have a continuous model with a convex, piecewise-linear objective, you should see a significant performance boost from using this feature. New attributes have been added to support this feature:
    • NumPWLObjVars: Indicates how many variables in the model have piecewise-linear objective functions.
    • PWLObjCvx: For each variable, indicates whether the corresponding piecewise-linear objective function is convex.
  • Support for models with more than 2 billion non-zeroes in the constraint matrix: You can now build and solve models whose constraint matrices have more than 2 billion non-zero values. This feature is transparent in our object-oriented interfaces, and mostly transparant in our C interface. We have added enhanced versions of a few existing C routines for cases where the change is visible (GRBXaddvars, GRBXaddconstrs, GRBXaddrangeconstrs, GRBXchgcoeffs, GRBXgetconstrs, and GRBXgetvars)
  • New explicit lazy constraint: Users can use the new Lazy constraint attribute to mark constraints as lazy. Users can also now indicate that constraints should be treated as lazy in LP and MPS files.
  • New asynchronous optimization methods: Users of the C and C++ APIs can now start optimization routines asynchronously, thus allowing programs to work on other things while optimization proceeds. Use the new GRBoptimizeasync and GRBModel::optimizeasync routines. These routines replace the previous NonBlocking parameter, which allowed asynchronous optimization in a Compute Server environment only.
  • New scaling option: Users can now select a new aggressive scaling option (ScaleFlag=2) when solving continuous models that are particularly numerically challenging.
  • Additional new attribute:
    • ObjBoundC: In contrast with ObjBound, this new model attribute gives you the unrounded MIP best bound.

Code Conversion from Version 5.6

Programs that were originally written for version 5.6 of the Gurobi Optimizer generally won't require any code changes to use version 6.0. If you'd like to use the new features, some code changes are required.
